# 客服 Copilot V1 实现版本文档

## 1. 文档信息
- 版本号：`v0.1.0-mvp`
- 日期：`2026-03-11`
- 模块归属：`agent-service`
- 目的：沉淀本次实现流程、代码改动和演进路线，方便后续回看与面试讲解。

## 2. 这个版本解决了什么问题
V1 聚焦客服场景中的最小业务闭环：
1. 客服输入用户问题。
2. 系统从知识库检索并生成“可编辑回复草稿”。
3. 当置信度不足时，不硬答，改为“澄清问题”或“建议升级工单”。
4. 全流程写入记忆与审计日志，保证可追踪。

这版的目标是先打通工程闭环和模块化架构，不追求模型效果上限。

## 3. 范围与边界
### 本版已实现
1. NestJS 模块化后端架构。
2. 知识库文章管理与检索。
3. 短期记忆（会话摘要）和长期记忆（客户事实）。
4. Copilot 回复决策策略：`reply / clarify / escalate`。
5. 工单创建与审计事件。

### 本版未实现
1. 外部 CRM / 工单系统真实对接。
2. 向量库检索与 Embedding 管线。
3. MiniMax 真实 LLM 调用适配器。
4. 完整权限体系（RBAC、多租户、细粒度鉴权）。

## 4. 架构落地概览
App 通过 `AppModule` 组合以下模块：
1. `DataStoreModule`

### Step 6：Copilot 核心编排
在 `CopilotService.draftReply` 实现核心链路：
1. 规范化用户问题。
2. 读取客户长期记忆 + 会话短期记忆。
3. 知识库检索 top3。
4. 组装 citations。
5. 计算 confidence。
6. 依据阈值决策动作：
   - `confidence >= 0.65` -> `reply`
   - `confidence < 0.65 且有命中` -> `clarify`
   - `confidence < 0.65 且无命中` -> `escalate`
7. 生成草稿、澄清问题或工单建议。
8. 更新会话摘要。
9. 写入审计事件 `copilot.draft_generated`。

API：
- `POST /api/copilot/draft-reply`

- `apps/agent-service/src/modules/copilot/copilot.module.ts`

## 6. 面试可复述的“单次请求链路”
可以按以下 8 句直接讲：
1. 客服端发起 `POST /api/copilot/draft-reply`。
2. Copilot 先取长期记忆（客户事实）和短期记忆（会话摘要）。
3. 然后去知识库做检索，取 top3。
4. 将命中结果加工成 citations，附标题、片段和分数。
5. 基于命中情况和问题复杂度计算置信度。
6. 置信度高走 `reply`，低则走 `clarify` 或 `escalate`。
7. 更新会话摘要，保持多轮连续性。
8. 写审计日志，保证可追踪和可复盘。

## 7. 相比初始模板的核心变化
1. `hello world` 单接口改为多模块业务 API。
2. 从单 Service 改为按领域拆分，支持后续替换真实基础设施。
3. 补齐了“检索 -> 决策 -> 升级 -> 审计”的业务闭环。
4. 引入低置信度兜底策略，避免 AI 硬答。
5. 提前准备了生产态数据库 schema 和接口文档。

## 8. 技术取舍（面试高频）
1. 先用内存仓储：降低环境复杂度，优先验证业务契约。
2. 再做向量化迁移：保留 controller/service 契约，底层替换 repository 即可。
3. 先规则决策再接 LLM：确保可解释、可回归测试。
4. 坚持 human-in-the-loop：客服审核后再发，先控风险。

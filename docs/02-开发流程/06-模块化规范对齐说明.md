# 模块化规范对齐说明

## 1. 对齐目标
按你给的要求，本项目规范对齐如下：
1. 前端模块化参考 `nextjs-template`。
2. 后端模块化与数据库变更管理参考 `42service`。

## 2. 前端规范（参考 nextjs-template）

## 2.1 目录分层
1. `src/app`：路由页面层。
2. `src/components/business`：业务组件层。
3. `src/components/common`、`src/components/ui`：通用组件层。
4. `src/services/modules`：按领域拆分 API 服务。
5. `src/hooks`：可复用 hooks。
6. `src/lib`：基础工具和底层能力。

## 2.2 约束
1. 页面尽量不写重逻辑，逻辑下沉到 hooks/service。
2. 组件按业务域拆目录，避免“大组件文件”。
3. 常量、类型、请求封装统一管理。

## 3. 后端规范（参考 42service）

## 3.1 模块分层
1. `src/modules/<domain>`：按业务域拆模块。
2. 每个域最少包含：`module/service/controller/dto`。
3. 跨域类型放 `src/modules/shared`。

## 3.2 数据库规范
1. 数据库连接与实体在 `src/database`。
2. schema 变更只允许通过 migration。
3. migration 历史统一保存在 `src/database/migrations`。
4. 禁止生产环境 `synchronize: true`。

## 3.3 配置规范
1. API Key、数据库连接、阈值都通过环境变量注入。
2. 启动阶段进行环境变量校验。
3. 禁止将敏感信息硬编码到代码仓库。

## 4. 当前已完成对齐项
1. 后端业务模块已按 domain 拆分。
2. 已引入 TypeORM + migration（PostgreSQL）。
3. 已建立 `.env + validate` 配置体系，移除脚本替换环境文件。
4. 已提供本地数据库一键启动脚本（docker compose）。

# 学习路径与执行步骤（按顺序）

> 建议按本文顺序学习，不要跳读。每一步都有：学习目标 + 操作动作 + 通过标准。

## 第 0 步：先把项目跑起来（必做）
- 阅读：`00-本地启动与联调手册.md`
- 学习目标：
1. 用最短路径跑通数据库、后端、前端和最小联调闭环。
- 操作动作：
1. 按手册执行 `pnpm install`、`db:up`、`db:run`、`start:dev`、`agent-desktop dev`。
- 通过标准：
1. `http://localhost:3006/health` 可访问，且 `draft-reply` 接口可返回数据。

## 第 1 步：先看产品定位
- 阅读：`../01-产品文档/01-产品架构设计.md`
- 学习目标：
1. 明确这不是聊天机器人，而是“客服 Copilot + 人工审核 + 工单升级”系统。
- 操作动作：
1. 画一版简化流程图（提问 -> 检索 -> 决策 -> 升级）。
- 通过标准：
1. 你能用 1 分钟说明系统边界。

## 第 2 步：看需求边界
- 阅读：`../01-产品文档/02-MVP需求文档.md`
- 学习目标：
1. 分清“当前要做什么”和“暂时不做什么”。
- 操作动作：
1. 把 API 清单抄成开发任务列表。
- 通过标准：
1. 你能说清每个接口对应哪个业务动作。

## 第 3 步：看实现全貌
- 阅读：`01-V1实现版本文档.md`
- 学习目标：
1. 理解从模板到数据库化 MVP 的完整过程。
- 操作动作：
1. 对照文档看 `app.module.ts` 和 `database.module.ts`。
- 通过标准：
1. 你能说出每个模块职责。

## 第 4 步：看改动明细（重点）
- 阅读：`04-改动明细与原因.md`
- 学习目标：
1. 清楚“为什么这么改”，而不是只知道“改了什么”。
- 操作动作：
1. 挑 3 个改动写成面试答题稿。
- 通过标准：
1. 每个改动都能解释：作用 + 原因 + 替代方案。

## 第 5 步：跑数据库 + 跑接口
- 阅读：`02-接口调用示例.md`
- 学习目标：
1. 跑通完整闭环。
- 操作动作：
1. `pnpm run db:up`
2. `pnpm run db:run`
3. `pnpm run start:dev`
4. 按文档依次调用 knowledge/copilot/tickets/audit。
- 通过标准：
1. 四段接口链路都返回正常。

## 第 6 步：理解迁移历史机制
- 阅读：`03-数据库设计.sql`
- 学习目标：
1. 理解当前 schema 与 migration 的对应关系。
- 操作动作：
1. 打开 `src/database/migrations` 对照字段。
- 通过标准：
1. 你能说明每张表在业务中的作用。

## 第 7 步：看规范对齐
- 阅读：`06-模块化规范对齐说明.md`
- 学习目标：
1. 明确前端/后端各自的模块化约束。
- 操作动作：
1. 对照 nextjs-template 检查前端目录规划。
- 通过标准：
1. 你能写出下一步前端重构计划。

## 第 8 步：进入向量检索阶段
- 阅读：`../03-进阶指南/01-pgvector安装与迁移步骤.md`
- 学习目标：
1. 从关键词检索升级到向量检索。
- 操作动作：
1. 设计 embedding 入库任务。
2. 替换检索 SQL。
3. 做混合检索打分。
- 通过标准：
1. 同义问法能稳定命中知识库。

## 第 9 步：面试复盘模板
最终你要能在 3 分钟讲清：
1. 业务价值：它解决客服什么问题。
2. 技术结构：模块如何拆，数据如何流。
3. 风险控制：低置信度为什么要澄清/升级。
4. 工程能力：为什么要 migration + 环境校验 + 可审计。

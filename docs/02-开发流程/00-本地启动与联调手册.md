# 本地启动与联调手册（小白版）

> 目标：第一次拿到项目，30 分钟内跑通「数据库 + 后端 + 前端 + 可选 MiniMax AI」。

## 1. 前置环境

请先安装以下工具：

1. Node.js `>= 22`（建议 22 LTS）
2. pnpm `>= 10`
3. Docker Desktop / OrbStack（用于本地 PostgreSQL + pgvector）
4. Git

可选检查命令：

```bash
node -v
pnpm -v
docker -v
docker info
```

## 2. 安装依赖

在仓库根目录执行：

```bash
pnpm install
```

## 3. 配置后端环境变量

后端位于 `apps/agent-service`，读取规则如下：

1. 优先读取 `.env.<NODE_ENV>`，例如 `.env.dev`
2. 再读取 `.env`

建议先复制示例文件：

```bash
cp apps/agent-service/.env.example apps/agent-service/.env.dev
```

本地默认配置如下：

```env
NODE_ENV=dev
PORT=3006

DB_HOST=127.0.0.1
DB_PORT=5432
DB_USER=root
DB_PASSWORD=zjf012511
DB_NAME=agent_service
DB_SCHEMA=public
DB_LOGGING=false
DB_SSL=false

RETRIEVAL_TOP_K=3

AI_CHAT_ENABLED=false
AI_CHAT_PROVIDER=minimax
AI_CHAT_MODEL=abab5.5-chat
AI_CHAT_TEMPERATURE=0.2
AI_CHAT_MAX_TOKENS=800
AI_CHAT_TIMEOUT_MS=15000

MINIMAX_API_KEY=
MINIMAX_GROUP_ID=
MINIMAX_BASE_URL=https://api.minimax.chat/v1
```

说明：
1. 默认 `AI_CHAT_ENABLED=false`，所以不填 MiniMax Key 也能先跑通项目。
2. 如果要开启 MiniMax，把 `AI_CHAT_ENABLED` 改成 `true`，并补齐 `MINIMAX_API_KEY` 与 `MINIMAX_GROUP_ID`。
3. 一旦开启 AI，启动阶段会做强校验，缺少密钥会直接报错，不会静默降级。

## 4. 启动数据库（PostgreSQL + pgvector）

在仓库根目录执行：

```bash
pnpm --filter agent-service db:up
```

查看数据库日志：

```bash
pnpm --filter agent-service db:logs
```

停止数据库：

```bash
pnpm --filter agent-service db:down
```

如果提示 `Bind for 0.0.0.0:5432 failed: port is already allocated`：
1. 说明本机已有其他 PostgreSQL 占用了 `5432`
2. 停掉原服务，或者修改 `docker-compose.db.yml` 映射端口
3. 同步修改 `apps/agent-service/.env.dev` 的 `DB_PORT`

## 5. 执行数据库迁移

在仓库根目录执行：

```bash
pnpm --filter agent-service db:run
```

成功后会创建核心表（如 `knowledge_articles`、`tickets`、`audit_events` 等）。

如果要回滚最近一次迁移：

```bash
pnpm --filter agent-service db:revert
```

## 6. 启动后端服务（NestJS）

在仓库根目录执行：

```bash
pnpm --filter agent-service start:dev
```

默认端口：`3006`。

健康检查：

```bash
curl http://localhost:3006/health
```

返回 `{"code":"00000","msg":"ok","data":{"status":"ok",...}}` 说明后端正常。

## 7. 启动前端服务（Next.js）

新开一个终端，在仓库根目录执行：

```bash
cp apps/agent-desktop/.env.development.example apps/agent-desktop/.env.development.local
pnpm --filter agent-desktop dev
```

默认访问地址：`http://localhost:3000`。

当前首页就是 Copilot 工作台，会直接请求 `agent-service`。

## 8. 最小联调（先验证规则版闭环）

### 8.1 创建知识库文章

```bash
curl -X POST http://localhost:3006/api/knowledge/articles \
  -H "Content-Type: application/json" \
  -d '{
    "title":"退换货政策",
    "content":"签收后7天内可申请退货，商品需不影响二次销售。",
    "tags":["售后","退货"]
  }'
```

### 8.2 生成客服回复草稿

```bash
curl -X POST http://localhost:3006/api/copilot/draft-reply \
  -H "Content-Type: application/json" \
  -d '{
    "conversationId":"conv_001",
    "customerId":"cus_001",
    "question":"我昨天签收，今天能退货吗？",
    "channel":"wechat"
  }'
```

如果能返回 `data.action / data.confidence / data.draftReply`，说明主链路已跑通。

## 9. 开启 MiniMax + LangChain 联调

当你已经能跑通规则版之后，再进行这一段。

### 9.1 打开后端 AI 配置

编辑 `apps/agent-service/.env.dev`：

```env
AI_CHAT_ENABLED=true
AI_CHAT_PROVIDER=minimax
AI_CHAT_MODEL=abab5.5-chat
MINIMAX_API_KEY=你的密钥
MINIMAX_GROUP_ID=你的GroupId
```

### 9.2 重启后端

```bash
pnpm --filter agent-service start:dev
```

### 9.3 再次调用草稿接口

```bash
curl -X POST http://localhost:3006/api/copilot/draft-reply \
  -H "Content-Type: application/json" \
  -d '{
    "conversationId":"conv_002",
    "customerId":"cus_002",
    "question":"我买的是企业采购订单，发票可以补开吗？",
    "channel":"wechat"
  }'
```

此时后端会走：
1. 知识检索
2. 短期/长期记忆读取
3. LangChain Prompt 编排
4. MiniMax 结构化 JSON 决策
5. 守卫逻辑兜底

## 10. 常见问题排查

### 10.1 `ECONNREFUSED 127.0.0.1:5432`

原因：后端连不上数据库。按顺序检查：

1. Docker 是否启动
2. `pnpm --filter agent-service db:up` 是否成功
3. 5432 端口是否被占用
4. `.env.dev` 的 DB 账号密码是否和容器一致

### 10.2 迁移报错 `extension "vector" ...`

原因：数据库不是 pgvector 镜像，或权限不足。  
处理：使用项目提供的 `docker-compose.db.yml` 启动数据库。

### 10.3 `MINIMAX_API_KEY 不能为空`

原因：你开启了 `AI_CHAT_ENABLED=true`，但没有填 MiniMax 凭证。  
处理：补齐 `MINIMAX_API_KEY` 和 `MINIMAX_GROUP_ID`，或者先把 `AI_CHAT_ENABLED=false`。

### 10.4 IDE 一片标红但命令行通过

处理顺序：

1. VSCode 执行 `ESLint: Restart ESLint Server`
2. VSCode 执行 `TypeScript: Restart TS Server`
3. 重新打开仓库根目录

## 11. 常用命令速查

```bash
# 根目录安装依赖
pnpm install

# 启动数据库
pnpm --filter agent-service db:up

# 跑迁移
pnpm --filter agent-service db:run

# 启动后端（开发）
pnpm --filter agent-service start:dev

# 启动前端（开发）
pnpm --filter agent-desktop dev

# 后端代码检查
pnpm --filter agent-service lint
pnpm --filter agent-service exec tsc --noEmit
```
